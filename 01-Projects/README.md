# 项目 (Projects)

本目录用于存放当前进行中的项目文档。

## 使用规范

### 1. 项目创建
- 使用`05-Templates/项目笔记模板.md`创建新项目
- 命名规范：`项目名称-简要描述.md`
- 及时填写项目概述、目标、时间线

### 2. 项目管理
- **状态跟踪**：定期更新项目状态（planning/in_progress/completed）
- **任务分解**：使用任务清单跟踪具体工作
- **周报更新**：每周记录进展、问题和计划
- **相关资源**：链接会议记录、技术决策、代码仓库

### 3. 项目归档
- 项目完成后，移动到`04-Archives/`目录
- 更新项目状态为"completed"
- 添加项目总结和经验教训

## 项目分类建议

### 按类型
- **产品开发**：新功能、产品迭代
- **技术优化**：性能提升、架构重构
- **团队建设**：流程改进、工具引入
- **战略规划**：技术路线图、AI融合

### 按优先级
- **高优先级**：业务关键、紧急修复
- **中优先级**：重要改进、季度目标
- **低优先级**：优化项、技术债务

## 关联关系

项目笔记应与以下内容建立链接：
- [[技术决策记录]] - 项目中的关键技术选择
- [[会议记录]] - 项目相关会议
- [[代码片段]] - 项目中的关键代码
- [[学习笔记]] - 项目中学到的知识
