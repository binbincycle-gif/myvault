# 收件箱 (Inbox)

本目录用于临时存放未处理的信息、想法、待办事项。

## 使用原则

1. **临时存放**：所有新信息先放入这里
2. **每日清空**：每天至少处理一次收件箱
3. **快速分类**：决定每项信息的最终归属
4. **避免堆积**：保持收件箱为空或接近空状态

## 处理流程

### 1. 收集
- 临时想法、灵感
- 待读文章、资料
- 待办事项提醒
- 会议记录草稿

### 2. 处理（每天）
对每项内容做出决定：
- **删除**：无用信息
- **归档**：已完成的事项 → `04-Archives/`
- **行动**：需要执行的任务 → 添加到相应项目
- **参考**：有价值资料 → `03-Resources/`
- **项目**：新项目想法 → `01-Projects/`
- **学习**：需要深入研究 → `03-Resources/AI学习/`

### 3. 组织
- 添加标签（如 `#todo`、`#waiting`）
- 建立相关链接（`[[相关笔记]]`）
- 设置提醒日期（如有需要）

## 示例文件命名
- `YYYYMMDD-简短描述.md`
- `待处理-主题.md`
- `想法-XXX.md`

---

*记住：收件箱不是存储区，而是处理中转站。*
