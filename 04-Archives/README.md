# 归档 (Archives)

本目录用于存放已完成的项目、过时的资料、历史记录。

## 归档原则

### 1. 什么应该归档
- ✅ 已完成的项目文档
- ✅ 历史版本的技术文档
- ✅ 过时的参考资料
- ✅ 已解决的bug记录
- ✅ 历史会议记录（超过1年）

### 2. 什么不应该归档
- ❌ 正在使用的模板
- ❌ 经常需要查阅的参考资料
- ❌ 团队共享的核心文档
- ❌ 仍在生效的政策流程

## 归档流程

### 1. 准备归档
- 确认内容已不再需要频繁访问
- 更新文档状态为"completed"或"archived"
- 添加归档日期和简要说明

### 2. 移动文件
- 将文件从原目录移动到`04-Archives/`
- 保持原有目录结构（如`04-Archives/01-Projects/项目名称/`）
- 更新相关笔记中的链接（如有必要）

### 3. 更新索引
- 在归档文件的frontmatter中添加：
  ```yaml
  archived: true
  archived_date: YYYY-MM-DD
  archived_reason: "项目已完成"
  ```

## 归档结构建议

```
04-Archives/
├── 01-Projects/          # 归档的项目
│   ├── 2025-Q1-项目A/
│   └── 2025-Q2-项目B/
├── 02-Technical/         # 技术文档归档
│   ├── 旧架构设计/
│   └── 历史方案/
├── 03-Meetings/          # 历史会议记录
│   └── 2024/
└── 04-References/        # 过时参考资料
    └── 技术书籍笔记/
```

## 检索建议

虽然归档文件不常访问，但偶尔可能需要检索：
- 使用Obsidian搜索功能
- 添加适当的标签（如`#archived`）
- 维护归档索引文件（可选）

## 定期清理建议

- **每季度**：回顾归档内容，确认是否需要永久保留
- **每年**：考虑将更早的归档压缩备份
- **根据需要**：彻底删除无价值的归档文件
